\begin{comment}
% Mention investigation into whether or not local choice and the proposed
% solution are equivalent.

It may be possible to encode local choice $P+Q$ by using a server process
$(x[inl].0 | x[inr].0)$ communicating with a client process $(\case{x}{P}{0} |
\case{x}{Q}{0})$. Such an encoding is clearly linear. However, in our current
type system, the problem with such an encoding is that we will have promised to
consume the full resources available in both client instances -- whereas one of
them would like to reduce to the inactive processes. In order to type this, we
would have to add a new typing rule, which would practically amount to adding
local choice.

The other way around it may be possible to encode the non-deterministic global
choice present in the cut on $!_nA$ and $?_nA$ using local choice by
constructing an arbiter process which receives $2n$ channels, and then has a
massive number of local choices to choose one possible way of linking them.
Assuming that local choice will choose either branch with equal probability,
however, we may have trouble constructing a faithful encoding of the global
non-determinism. Furthermore, the encoding is cumbersome -- due to the presence
of an arbiter process -- and unpleasant. Some of this trouble can be done away
with by generalizing local choice to an arbitrary number of parameters -- or by
being content in the knowledge that we can only pleasantly encode 2-ary
non-determinism.
\end{comment}
